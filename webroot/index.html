Microsoft VSTS Work Item Migrator

Ett enkelt open source cross platform verktyg som ska funka på både TFS och Azure DevOps. Det har begränsade konfigurationsmöjligheter dock (man kan inte konfigurera varken mall eller mappa work item typer eller fält) och det är oklart om det funkar att migrera ett Scrum-projekt till ett Agile-projekt, jag har inte fått det att funka och jag har inte hittat varken dokumentation eller konfigurering som stödjer det. Jag har inte heller lyckats migrera ett Scrum-projekt till ett annat Scrum-projekt utan att work items ignoreras och det finns ingen vidare dokumentation eller vettig applikationsinformation heller så jag vet inte varför migreringen tappar eller inte hittar alla work items. Det finns inget färdigkompilerat program att köra utan man måste ladda ner källkoden och ställa sig i rätt mapp med ett cmd line tool. Koden använder REST-API:erna på ett bra sätt och kör .NET Core vilket känns modernt.

Pros:

Byggt på .NET Core så det är plattformsoberoende Använder REST-API:erna

Cons:

Inget färdigkompilerat program att använda Begränsade konfigurationsmöjligheter Jag har inte fått det att funka fullt ut Migrerar inte mellan templates (?) Undermålig dokumentation

nkdAgility Azure DevOps Migration Tools

Ett kraftfullt open source verktyg som funkar på både TFS och Azure DevOps. Man kan konfigurera väldigt mycket, nästan för mycket så det är en hel del att sätta sig in i innan man får till en konfigurering som funkar att köra, men när man väl börjar lära sig hur konfigurationen funkar är det väldigt användbart och det funkar bra att migrera ett Scrum-projekt till ett Agile-projekt t ex. Man kan göra väldigt avancerade filtreringar och regex-uttryck i konfigureringen vilket gör verktyget otroligt kraftfullt och flexibelt. Det avråds att använda detta verktyg för migrering av testplaner och testsviter dock.

Pros:

Kraftfullt och flexibelt Otroligt konfigurerbart Enkel installation via Chocolatey Bra dokumentation Sparar analytics i Application Insights för bättre felsökning och tracking (valbart) Migrerar mellan olika templates

Cons: Alla konfigurationsmöjligheter gör en lite förvirrad i början Migrering av testplaner och testsviter funkar inte trots att konfigurationen för det finns

Solidify Work Item Migrator

Ett verktyg som har lite begränsning i att man inte kan migrera ett Scrum-projekt till ett Agile-projekt eftersom man inte kan konfigurera varken mall eller mappa work item typer eller fält och det känns lite hackigt i och med att man först måste skapa en lokal databas med hjälp av script. Det går däremot att migrera testartfakter även om det finns andra verktyg som ska fungera smidigare. För en 1:1 migrering så funkar verktyget bra.

Pros:

Funkar smidigt för 1:1 migreringar Migrerar testartefakter

Cons:

Ingen konfigurering direkt Migrerar inte mellan olika templates Man måste sätta upp en databas för migreringen

Solidify Jira/Azure DevOps Migrator

Detta verktyg funkar inte idag mellan TFS/Azure DevOps och TFS/Azure DevOps men det går att göra en export-funktion mot TFS/Azure DevOps istället för mot Jira och då kan man återanvända de konfigureringsmöjligheter som finns för mappning av work item typer och fält samt import-funktionen som redan går mot TFS/Azure DevOps. Det borde inte vara ett jättejobb att göra en ny export-funktion TFS/Azure DevOps iom att det finns verktyg som redan gör detta som man kan snegla på. Däremot bör man uppdatera projektet så att man kör REST-API:erna istället för SDK:erna för en modernare feel. Verktyget fokuserar på work item migrering och är således inte testat med testartefakter.

Pros:

Färdigkompilerat program Migrerar mellan olika templates Bra dokumentation Konfigurerbart

Cons:

Bygger på SDK:er som blir utdaterade Viss mappning sker i kod vilket gör det mindre flexibelt

Conclusion

Jag skulle nog själv välja nkdAgility-verktyget pga dess kraftfullhet, flexibilitet och konfigurerbarhet. Det känns som ett stabilt verktyg där vi förmodligen skulle kunna bidra med en del på GitHub. Det skulle även vara intressant att göra en TFS/Azure DevOps-variant av Jira-exportprogrammet så man skulle kunna bygga upp en svit av exportprogram som vi kan ha tillsammans med importprogrammet till TFS/Azure DevOps. Alternativet skulle vara att uppdatera vårt TFS-verktyg men det känns som att Jira-verktyget har en bättre grund om man ändå måste utveckla något. Men för ett projekt som behöver migreras idag behövs ett färdigt verktyg och då skulle jag rekommendera nkdAgility Azure DevOps Migration Tool.
